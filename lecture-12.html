Transformer & DNN Training Accelerators

Training is not just a bigger forward pass. It adds a backward pass, transposed matmuls, nonlinear operations the systolic array can't do, and a huge activation memory bill. This note walks through the hardware that attacks each of these.

NYU SAI Lab Spring 2026 Systolic arrays · BFP backprop · softmax · LayerNorm
01

The training workload: it isn't all matmul

A trained accelerator has to do three things inference never asked of it: run the backward pass, feed nonlinear and transposition operators, and hold the activations alive until the gradients arrive. Start by seeing where the time actually goes.

The self-attention block forward recap

With batch B, sequence L, embedding E. From each input token, three vectors — Query Q, Key K, Value V — are produced by linear projections, then scores are formed, softmaxed, and applied to the values:

Q,K,V: (B,L,E) × (E,E) → (B,L,E)three linear projections
QKᵀ: (B,L,E) × (B,E,L) → (B,L,L)pairwise scores
softmax(QKᵀ)·V: (B,L,L) × (B,L,E) → (B,L,E)scale → softmax → context

Scale, softmax-normalize, multiply the values, pass through the output linear, and add the residual. Notice the non-matmul steps sprinkled through: a transpose to form Kᵀ, a softmax, and (per block) two LayerNorms.

Where does the latency go?

Measured latency breakdown on two LLMs — matmul dominates but is far from the whole story:

ModelMatmulNormalizationSoftmaxOthers (e.g. transpose)
GPT-252.2%16.1%13.9%17.8%
OPT57.2%14.9%13.4%14.5%
Why this matters for hardware

Matmul is the majority, but the nonlinear operations are not negligible — normalization + softmax + "others" together are ~45%. A systolic array only speeds up the matmul; if the LayerNorm, softmax, and transposes run slowly on a general core, they become the bottleneck by Amdahl's law. Hence Part 2: give the non-matmul ops their own hardware.

Tiling the matmul

Large operands don't fit the compute core, so they are partitioned into tiles sized to the array (e.g. a 512×256 · 256×256 product broken into 128×128 tiles A00, B00, …). Each tile pair is one systolic-array pass; results accumulate across the shared dimension.

02

Hardware for operations other than matmul

Everything the matmul engine can't do: transposition, softmax, division, LayerNorm, GeLU. The recurring trick is to turn expensive operators (exp, reciprocal, √) into shifts and adds in the log domain, or to skip them entirely.

In-place matrix transposition

Transpose a matrix within its existing memory, using only minimal extra storage. A row-major array a b c d e f g h (2×4) must become the column-major order a e b f c g d h (4×2). The elements move in permutation cycles:

(b,c,e) → (e,b,c) · (d,f,g) → (f,g,d)a and h are fixed points
  • Logic must read multiple entries, permute them, and write them back — done efficiently with minimal memory-access cost.
  • Finding the swapping pattern that minimizes permutations is a classic math problem — Brenner, Algorithm 467 (ACM TOMS, 1973).

Softmax exp via shift-add

Softmax is everywhere in transformers. The costly parts are the exponential and the division:

si = ezi / Σj=0N−1 ezjfor i = 1…N

Rewrite the exponential in base 2 so it becomes a shift-and-multiply. For positive z in INT representation:

ez = 2z·log₂e = 2u+v , log₂e ≈ 1.0111₂change of base
z·log₂e ≈ z + (z>>2) + (z>>3) + (z>>4)multiply-by-constant → shifts
ez = 2u+v ≈ 2u·(1 + v/2)u = exponent, v/2 = mantissa
The pattern

u is the integer part of the exponent (becomes a bit-shift) and v is the fractional part; v/2 serves as the mantissa. So 2u(1+v/2) is a shift plus one small multiply — no exp unit needed. This is the Taylor approximation 2v ≈ 1 + v/2 for small v (Xia & Zhang, "Softmax Acceleration," 2023).

Division reciprocal in log domain

Floating-point division collapses to add/subtract in the log domain, again using log₂(1+m) ≈ m:

a/b = 2ea(1+ma) / 2eb(1+mb) ≈ 2ea−eb+ma−mbFP division → subtract exponents

For INT operands, a conventional hardware divisor (accumulator + n+1-bit adder + shift-left quotient logic) is used instead.

LayerNorm inverse √ via log

Normalization needs a mean, a variance, and — the costly bit — the inverse of the standard deviation:

s = α·(z−μz)/σz + βμz = Σzi/N , σz = √(Σ(zi−μz)²/N)

The reciprocal square root is again a log-domain trick — halve the log and negate:

y = 1/√x ⇒ log₂y = −½·log₂xwith log₂x ≈ Ex − Q + Mx/2L + σx

Q is the bias; Ex, Mx are the binary exponent and mantissa. For functions too complex for these tricks (e.g. GeLU), fall back to a table lookup y = F(x) — precompute and store results — at the cost of extra memory access and footprint.

HAAN: skipping LayerNorm work accelerator

HAAN exploits a measured fact: the inverse standard deviation (ISD) of the input is correlated across layers — in deep blocks it becomes nearly linear in log scale.

  • Exploit correlation in input statistics across layers → skip redundant computation and estimate the normalization statistics with a lightweight log-linear predictor.
  • Datapath: Input Statistics Calculator (parallel mean/variance) → Square Root Inverter (Newton's method, with layer-skipping support) → Normalization Unit.

Peng et al., "HAAN: A Holistic Approach for Accelerating Normalization Operations in LLMs," 2025.

PICACHU: a reconfigurable nonlinear engine CGRA

PICACHU is a plug-in coarse-grained reconfigurable array (CGRA) tailored to nonlinear ops, using custom algorithms plus a dedicated compiler toolchain.

  • Decompose: every LLM nonlinearity breaks into a few math operators — Softmax/GeLU/SwiGLU need division + exponential, LayerNorm/RMSNorm need inverse √, RoPE needs sine/cosine.
  • Heterogeneous tiles: Branch-optimized (BrT), Basic (BaT), and Compute (CoT) tiles, each with distinct functional units, plus Precision-Aware FUs (vector adders/multipliers reconfigurable across bit-widths).
  • Compiler does pattern matching & offloading (e.g. fmul…fadd ⇒ GeLU), emitting @cgra_SOFTMAX, @cgra_LAYERNORM calls.

Qin et al., "PICACHU: Plug-In CGRA Handling Upcoming Nonlinear Operations in LLMs," ASPLOS 2025.

03

Hardware for the backward pass

The backward pass is the same matmul engine run with transposed operands. The key hardware insight: you never build a separate transpose unit — you feed the same systolic array from a different edge.

Backprop is two transposed matmuls

For a linear layer Y = X·W (with X: L×E, W: E×E), the backward pass computes an input/activation gradient and a weight gradient:

∇X = ∇Y · Wᵀactivation gradient (flows to prev layer)
∇W = Xᵀ · ∇Yweight gradient (used by optimizer)

The same structure holds for a 1×1 convolution viewed as a matmul with X: BHW×C, W: C×N:

PassOperationShapes
ForwardY = X·W(BHW×C)·(C×N) = (BHW×N)
Activation grad∇X = ∇Y·Wᵀ(BHW×N)·(N×C) = (BHW×C)
Weight grad∇W = Xᵀ·∇Y(C×BHW)·(BHW×N) = (C×N)
In-place transposed matmul

Training needs transposed matmuls, but building dedicated transpose hardware is wasteful. Instead, transposition is absorbed into the systolic array's dataflow — the transpose is realized by which edge you inject operands and which edge you read results from. One array, three dataflows.

Three dataflows on one systolic array

Reusing the same weight-stationary array, only the injection/read directions change:

ComputationStationaryInput feeds fromOutput read at
Forward X·WWeights preloadedBottom (streams up)Right
Activation grad ∇Y·WᵀWeights preloadedLeft (data gradient)Top (accumulate upward)
Weight grad Xᵀ·∇YAccumulation-stationary∇Y from left, X from bottomStored in cells

For the weight gradient the products accumulate and stay inside the systolic cells — exactly the accumulation-stationary mode below.

Two flavors of systolic cell

A systolic cell is a MAC that passes data to its neighbors. Which value it "keeps" defines the dataflow:

Weight-stationary

z = w·x + y ; v = x. The weight w stays in the cell; data x,y flow through. Used for forward and activation-gradient passes.

Accumulation-stationary

z = y ; v = x ; q = x·y + q. The running sum q stays in the cell (stored in the MAC). Used for the weight-gradient pass.

Worked accumulation-stationary example: with y=2, x=4, the cell passes z=2 (old y) and v=4 (old x) to neighbors while updating q = 4·2 + 0 = 8 locally — the partial sum never leaves the cell until the tile finishes.

04

Reduced-precision training: BFP & FAST

Training in FP32 is expensive. The trick is block floating point (BFP) plus a schedule that starts low-precision and grows precise — fast first, accurate second.

Block floating point dot product

In BFP, a group of values shares one exponent; only the mantissas differ. A dot product multiplies the shared exponents once and sums the mantissa products:

(2² · [+14,+2]) · (2⁴ · [+4,−9]) = 2⁶ · (14·4 + 2·(−9)) = 2⁶ · [+38]exponent add + mantissa MACs
Why BFP beats plain FP

Ordinary FP does an exponent addition and mantissa alignment for every element. BFP does them only once per group — the per-element work is just integer mantissa multiplies. Cheaper hardware, most of the dynamic range preserved.

FAST: Fast First, Accurate Second Training

Two observations from prior work motivate a precision schedule:

  • Adding zero-mean Gaussian noise to the weight gradient ∇W reduces overfitting and improves convergence.
  • Decreasing the noise variance over iterations beats a fixed noise level throughout training.
  • Hypothesis: the same effect can be had by adjusting the BFP precision of weights, activations, and gradients from low to high over training — low precision is the noise.

FAST progressively raises BFP precision along two axes — training iteration and layer depth:

SchemeEarly / shallowLate / deep
Temporal High-to-LowFP32 for weights/acts/grads (first part)Lower-precision BFP (second part)
Layerwise High-to-LowFP32 for the first ~10 layersLower-precision BFP for later layers
FAST = progressive precision

Move in the opposite direction of the naive scheme: start low precision (cheap, noisy, fast) and increase mantissa bit-width across both layer depth and iterations. On ResNet-18/ImageNet the per-layer (W,A,G) precision visibly climbs from (2,2,2) early to (4,4,4) late.

FAST system design

  • Systolic array of fMACs — FAST multiply-accumulate units that operate on chunks of BFP mantissas (e.g. 2-bit chunks), so mantissa width is variable in 2-bit increments.
  • BFP converters, an accumulator + systolic-array input generator, and a memory subsystem with separate Weight / Data / Gradient SRAMs.

Evaluation — Time-to-Accuracy

Using Time-to-Accuracy (TTA) to hit 68% on ResNet-18/ImageNet, FAST reaches the target fastest across all numeric formats (normalized training time, lower is better):

FormatFAST-AdaptiveMidBFPMSFP-12INT-12bfloat16Nvidia MPFP32
Normalized TTA1.001.862.272.923.855.698.51

Zhang, McDanel & Kung, "FAST: DNN Training under Variable Precision Block Floating Point with Stochastic Rounding," HPCA 2022.

05

Memory-efficient training

Backprop must keep every layer's input activations alive until its gradient arrives — so activation memory, not weights, dominates training storage. Two ideas fight this: don't store activations (reversible nets) and don't train everything (Duplex).

Activations dominate the footprint

  • In the forward pass each layer's output must be retained; in the backward pass those stored activations are consumed to form gradients — so memory grows proportional to depth.
  • Normalized training-storage (weights + activations) is overwhelmingly activation: ResNet-18 1× → ResNet-34 1.67× → ResNet-50 6.6× → VGG-16 13.4×.
  • Edge devices have small on-chip storage → frequent, costly off-chip memory access.

Reversible networks RevNet

A reversible residual network splits the input into (x₁, x₂) so that the inputs can be reconstructed from the outputs — no need to store activations:

Forward: y₂ = F₁(x₁) + x₂ , y₁ = F₂(y₂) + x₁two coupled residual blocks
Backward: x₁ = y₁ − F₂(y₂) , x₂ = y₂ − F₁(x₁)recompute inputs from outputs
The memory win

Because inputs are recoverable, the backward pass runs in three steps — recompute the input, compute the input gradient, then compute & apply the weight gradient — without ever storing the input activations. Memory is traded for a bit of recomputation.

Duplex DNN train a subset

Reversibility raises compute demand. Duplex answers with a frozen backbone + trainable branch:

  • A pretrained backbone DNN is frozen during the backward pass — only a small reversible branch DNN is trained, minimizing training cost.
  • Normalization layers are removed from the branch DNN to facilitate (and simplify) training.

Zhang et al., "CAMEL: Co-Designing AI Models and eDRAMs for Efficient On-Device Learning," HPCA 2024.

★

Exam recap

Training ≠ inference

Adds backward pass, transposed matmuls, and long-lived activations. Matmul is ~52–57% of latency; nonlinear ops are the rest.

Non-matmul tricks

exp, ÷, 1/√ all become shift-and-add in the log domain via 2ᵛ ≈ 1+v/2 and log₂(1+m) ≈ m. Else: table lookup.

HAAN / PICACHU

HAAN predicts LayerNorm stats across layers to skip work. PICACHU is a CGRA that decomposes nonlinearities into math operators.

Backprop math

∇X = ∇Y·Wᵀ (activation grad), ∇W = Xᵀ·∇Y (weight grad). Two transposed matmuls.

One array, 3 dataflows

Transpose is free — feed the systolic array from a different edge. Weight-stationary vs accumulation-stationary cells.

BFP

Shared exponent per group → exponent add once, not per element. Cheaper than FP with similar range.

FAST

Progressively raise BFP precision over depth + iterations. Low precision = gradient noise. Lowest TTA (1.00 vs FP32's 8.51).

Memory-efficient

Activations dominate (VGG-16: 13.4×). RevNet recomputes inputs; Duplex freezes the backbone, trains a branch.
