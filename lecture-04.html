Pruning Strategies for Efficient Deep Neural Networks

Making networks smaller and faster by zeroing out what doesn't matter — the compute & storage math, how to encode sparsity, which weights to cut, and how it all changes for transformers.

NYU SAI Lab Spring 2026 CNNs · Transformers MACs · sparsity · rewinding
01

Why prune?

Two payoffs: less compute at run time and less storage for the model. Everything downstream is a way to realize one of these.

The cost of a convolution

A conv layer maps input feature maps C×H×W through M filters of size C×R×S to output maps M×E×F. Each output pixel costs R×S multiply-accumulates (MACs) per input channel.

MACs = M×C×R×S×E×Fcompute per layer
Storage = 32 × (MCRS + CHW + MEF) bitsweights + in-act + out-act, FP32
Key distinction

Weights are persistent — they must be stored for the life of the model. Activations are transient — they can be freed the moment the layer finishes. This is why weight compression dominates the storage story.

What sparsity buys you

If a fraction p of weights are zero, you simply skip those MACs. If additionally a fraction q of inputs are zero, more MACs vanish — but only when a nonzero weight meets a nonzero input, hence the inequality.

weight-sparse: (1−p) × M·C·R·S·E·Fexact
+ input-sparse: ≥ (1−p)(1−q) × M·C·R·S·E·Flower bound
Offline vs online

Weight pruning is offline — done once, the sparsity pattern is fixed and free at inference. Activation pruning is online — the zero pattern depends on the input, so it must be recomputed every forward pass, adding sorting / selection overhead.

The cost of a transformer per block

With batch B, sequence length L, embedding E. The self-attention (SA) block spends most of its cost on the Q/K/V and output projections (E² terms) plus the two L² score/context matmuls:

SA = 4BLE² + 2BL²E4 linears + QKᵀ + ·V
FFN = 8BLE²E→4E, 4E→E
Block total = 12BLE² + 2BL²ESA + FFN
Read the two terms

The BLE² term (linear projections) dominates at short sequences; the BL²E attention term dominates at long sequences. Pruning weights attacks the first; pruning tokens attacks the second — which is exactly why transformers get their own pruning axis (§5).

Under sparsity the linear layer becomes (1−p)·BLE² nonzero MACs, or ≥(1−p)(1−q)·BLE² once input activations are sparse too — the same structure as convolution.

02

Storing sparsity: encoding schemes

A zeroed weight only saves memory if you don't store the zero. Each scheme trades index overhead against how much sparsity it assumes. Pick the encoding to match the sparsity level.

The catch

To compress you must encode the nonzeros and their positions. For activations this encoding itself costs compute every forward pass — another reason activation sparsity is harder to exploit than weight sparsity.

03

Which weights to cut? Pruning criteria

Every criterion is an importance score. Cut the low-scoring weights. The schemes differ in how much they know about the loss.

Magnitude pruning simplest

Keep a weight if its magnitude clears a threshold a, else zero it. The mask is:

mi = 1 if ‖wi‖₁ ≥ a , else 0threshold mask
Drawback

Magnitude ignores the input. A small weight that always multiplies a large, frequent activation can matter more than a large weight that rarely fires — magnitude can't see this.

Training-loss change Taylor / OBD

Better principle: cut the weight that changes the loss least. Taylor-expand the loss around the trained weights (pruning means w′ = 0):

ℒ(0) − ℒ(w) = (dℒ/dw)·w + ½·(d²ℒ/dw²)·w²2nd-order
  • First-order: sort weights by the product of gradient × value — cheap, no Hessian.
  • For a converged net the gradient ≈ 0, so the first term vanishes and the Hessian (curvature) term drives the decision — this is Optimal Brain Damage (LeCun et al., 1989).
  • The full Hessian is intractable; approximate it with the Empirical Fisher (outer product of gradients): H ≈ (1/N)·Σ gᵢgᵢᵀ.

Regularization-based L1 / Lasso

Don't prune after the fact — make training prefer sparse solutions by adding a penalty:

min ℒ(w) + λ‖w‖₁Lasso → sparse
Why L1 and not L2

The L1 ball has corners on the axes, so the optimum tends to land on an axis where coordinates are exactly 0. Mechanistically: the gradient of |w| is sign(w) — a constant push toward zero regardless of size — whereas w²'s gradient 2w fades near zero and never quite kills the weight. L2 shrinks; L1 zeroes.

The backward pass of pruning

Pruning is a masking node in the graph. The extra factor is ∂W′/∂W = 0 if pruned, else 1. So during backprop the gradients of pruned elements are masked out — they stay dead and don't receive updates.

04

A taxonomy: five ways to slice pruning

The same network can be described along five independent axes. Any real method is a point in this five-dimensional space.

AxisOption AOption B
WhenZero-shot (train → prune)Iterative (train ⇄ prune)
ShapeUnstructuredStructured · N:M
WhatWeight pruningActivation pruning
AdaptivityStatic (fixed pattern)Dynamic (per-input)
PurposePrune for inferencePrune for training

When to prune — iterative vs zero-shot

  • Zero-shot (a.k.a. post-training): train fully, then prune once. Fast, cheaper.
  • Iterative: loop prune a bit → retrain → repeat until the target sparsity is met. Best accuracy, but multiple training rounds cost compute.
Lottery Ticket Hypothesis · Frankle & Carbin, 2018

A randomly-initialized dense net contains a sparse subnetwork ("winning ticket") that — trained in isolation from the original init — can match the full network's accuracy in the same number of iterations.

  • Iterative Magnitude Pruning (IMP) finds the ticket: train → prune p% → rewind the survivors to their w₀ values → retrain. Repeat.
  • Weight rewinding: for larger nets, rewinding to an early checkpoint wᵢ (small i) beats rewinding all the way to w₀.
  • Provocative takeaway (Liu et al., "Rethinking the Value of Pruning"): often it's the discovered architecture that matters, not the inherited "important" weights — pruning as a form of architecture search.

Shape — unstructured vs structured vs N:M

Unstructured

Any weight anywhere. Best accuracy, but irregular pattern needs special hardware to speed up.

Structured

Whole vectors / kernels / filters. Hardware-friendly regular sparsity, lower accuracy.

N:M

Exactly N nonzeros per M. The middle ground — 2:4 runs on NVIDIA GPUs.

Granularity runs from fine-grained (0-D) → vector (1-D) → kernel (2-D) → filter (3-D). As you go coarser, hardware gets happier but accuracy drops — filter pruning falls off fastest.

Two structured tricks worth remembering

Network Slimming: attach the BatchNorm scaling factor γ to each filter and put an L1 penalty on those γ's during training — unimportant filters get scaled toward zero and are removed for free.

BatchNorm folding: at inference BN is affine, Y = p·X + q, so p folds into the conv weights and q into the bias — the BN layer disappears at run time.

Cascade effect

Pruning a filter in layer l deletes a whole output channel → that becomes a zero input channel in layer l+1 → which lets you drop the matching weights there too. Filter pruning propagates. (Holds when the BN bias is zero; otherwise the downstream map carries a constant.)

N:M in hardware: a 2:4 tensor keeps 2 of every 4 weights, stores them compressed with 2-bit indices, and feeds a small MAC array that multiplexes the right inputs — regular enough to accelerate, sparse enough to help. Supported on NVIDIA V100+.

What — weight vs activation pruning

Activation pruning is tempting (ReLU already makes activations ~50% zero) but hard:

  • Computing importance scores per input is expensive (it's online).
  • ReLU sparsity is irregular — the zero pattern shifts every sample.
  • Fix: impose a 2:4 pattern on the ReLU output, then row-compress — regular sparsity you can actually exploit (JointNF, Zhang et al., 2024).

Adaptivity — static vs dynamic

  • Static: one fixed pattern, components permanently removed.
  • Dynamic: keep the full network but skip unimportant components per input, chosen at run time by a lightweight saliency predictor. Preserves structure, adapts to each sample.

Purpose — pruning for training

Pruning isn't only for inference. SDGP (Structured Data Gradient Pruning) applies 2:4 sparsity to the gradient / data flowing backward (dL/dO × Wᵀ), speeding up training itself.

05

Transformer-specific pruning

Transformers open pruning dimensions CNNs don't have: you can drop tokens (sequence length L) and drop heads.

First, a useful fact

Splitting attention into M heads (each of dim D = E/M) does not change the total compute — M·L²·D = L²·E. Multi-head is a reshaping, not extra cost. So the win from head pruning is removing heads entirely.

Token pruning attacks the L² term

Not every token needs to survive every layer. Score each token's importance (e.g. average attention it receives) and drop those below a threshold — progressively, layer by layer.

s(l)(xᵢ) = (1/Nh)(1/n) Σh Σj A(h,l)(xᵢ,xⱼ)token importance
  • Removed tokens cut compute for every following layer — the savings compound with depth.
  • SpAtten (Wang et al., 2021) prunes tokens and heads jointly in a cascade — accumulate importance scores, keep top-k.
  • Intuition from sentiment example: by layer 12 only 2 tokens ("best," "returning") remain and still classify correctly.

Head pruning a new dimension

Reshape input (1, 197, 768) → (1, 12, 197, 64) and simply keep fewer heads → (1, 4, 197, 64). Which heads to cut? Attach a mask variable ξh ∈ {0,1} to each head and score by loss sensitivity:

Ih = 𝔼x∼X |∂ℒ(x)/∂ξh|head importance
Surprising result · Michel et al., "Are 16 Heads Really Better Than One?"

Most attention heads can be removed with little accuracy loss — a few specialized heads do the heavy lifting. In some cases removing a head actually raises BLEU/accuracy.

★

Exam recap

Two goals

Reduce compute (skip MACs) and reduce storage (don't store zeros). Weights persist; activations are transient.

Sparsity math

Weight-sparse = (1−p)·MACs (exact). +Input-sparse ≥ (1−p)(1−q)·MACs.

Transformer cost

Block = 12BLE² + 2BL²E. Linears scale E², attention scales L².

Encodings

Bitmap (low) → RLE → CSR (high) → COO (extreme). Match to sparsity level.

Criteria

Magnitude (ignores input) → Taylor/OBD (Hessian; grad≈0 when trained) → L1/Lasso (zeroes, not shrinks).

Lottery Ticket

Sparse subnet + right init matches dense. Find it with IMP + weight rewinding.

Structured trade

Unstructured = accuracy; structured = hardware. N:M (2:4) is the GPU-supported middle.

Transformer extras

Prune tokens (cuts L², compounds with depth) and heads (most are redundant).
