CNN & Transformer Accelerators

How specialized hardware runs deep networks fast and cheap — the accelerator template, systolic arrays for matmul, packing sparsity and terms into the array, the classic CNN chips, and what changes when the workload becomes a transformer.

NYU SAI Lab Spring 2026 Systolic arrays · TPU Column combining · Term quant. Diannao · Eyeriss · Cnvlutin
01

The AI-accelerator template

An AI accelerator is a co-processor that executes the part of the machine code tied to the AI workload — mostly big matrix multiplies. It sits beside the CPU and GPU on a NoC, sharing DRAM.

Anatomy of the chip

Every design in this lecture is a variation on the same block diagram: a small on-chip memory hierarchy feeding a dense arithmetic core.

  • Compute core — a Matrix MAC (2-D grid of multiply-accumulators) for matrix multiplication, plus a Vector MAC and a Special Function Unit for the non-matmul operations (activations, normalization).
  • Weight SRAM — holds the persistent weights streamed in from DRAM.
  • Activation / Data SRAM — holds transient input and output feature maps.
  • Register file + accumulator — staging and partial-sum accumulation, steered by control bits through a memory controller.
Why a separate accelerator at all

General cores waste energy on instruction fetch, control, and cache coherence. An accelerator spends its area on arithmetic and local SRAM, so the same silicon does far more MACs per joule — the entire field is about keeping that MAC array fed without paying DRAM energy on every operand.

Memory access reduction the central idea

Consider a 2×2 matmul W·X = Y. A weight such as W₀₀ is loaded once from Weight SRAM and held in the compute core; inputs stream past it from Activation SRAM and each product accumulates into the running Y.

Y₀₀ = W₀₀X₀₀ + W₀₁X₁₀ , Y₀₁ = W₀₀X₀₁ + W₀₁X₁₁2×2 product
The lever

The computation order and memory-access pattern can be rescheduled to minimize cost without changing the result. Reusing an operand already on-chip — instead of re-fetching it from DRAM — is the single biggest energy win, because a DRAM access can cost hundreds of times more energy than a MAC.

Spatial reuse & double buffering

A spatial architecture lays many processing elements (PEs) in a grid so an operand fetched once from memory can be reused across the whole row or column of PEs — great reuse, reduced memory-access cost.

Double buffering (ping-pong)

Split the input buffer into Mem₁ and Mem₂. While the systolic array consumes one buffer, DRAM fills the other; then they swap. This overlaps data production and consumption, hiding DRAM latency so the array never stalls waiting for operands.

02

Systolic arrays

The workhorse matmul engine. Coined by Kung & Leiserson (1978) and Kung (1982), it is a 2-D grid of MACs through which data flows rhythmically — "systolic," like a heartbeat. Google's TPU (2017) is a systolic array.

The systolic cell weight-stationary

Each cell holds one weight w and does two things per beat: a multiply-accumulate, and a pass-through of its input to the neighbor above.

z = w·x + y , v = xMAC + forward
  • Inputs: data x enters from below, partial sum y enters from the left.
  • Weight w stays resident in the cell's MAC — hence weight-stationary.
  • Outputs: the updated partial sum z exits right; the data v = x exits up to the next row.

Worked cell: with w = 6, x = 4, y = 2 → z = 6·4 + 2 = 26 flows right, v = 4 flows up.

Why weight-stationary wins

Weights are loaded once and reused for every input in the batch. Only activations and partial sums move — the highest-volume operand (weights) pays zero repeated fetch cost. This is the reuse principle of §1 made physical.

Inside the cell: the bit-serial MAC

The MAC can be built bit-serially: a bit-serial multiplier-accumulator feeds each weight bit W₇…W₀ through an AND gate against the streamed input Xᵢ, summing partial products through a chain of full adders with enable/reset control. Small area, one bit per cycle — a natural fit for the term-level tricks in §4.

Skewed dataflow: a worked matmul

Compute W · X with

W = [[2,−1,1],[−2,3,−4]] , X = [[1,0],[0,3],[−2,2]] , Y = [[0,−1],[6,1]]2×3 · 3×2
  • Weights preloaded (shown in red) into the array, one per cell.
  • Data enters from the bottom in a skewed (staggered) schedule — each column is delayed one beat relative to its neighbor so operands meet their partial sums at the right time.
  • Partial sums march left-to-right; as each column's inputs sweep through, a completed output element falls out of the right edge.
Why the input is skewed

All cells fire on the same clock, but the data destined for cell (i, j) must arrive exactly when the partial sum from (i, j−1) does. Staggering the inputs by one cycle per column aligns every operand with its accumulator — no explicit scheduling logic needed, the geometry does it.

03

Packing sparsity: column combining

Pruning zeros out small weights — but a systolic array is a dense grid, so a zeroed cell still occupies silicon and a clock cycle. Column combining (Kung, McDanel & Zhang, ASPLOS 2019) makes that sparsity pay off.

The problem

After pruning, the weight matrix is sparse and irregular. It is hard to leverage those zero weights to reduce hardware cost — a naive systolic array runs the zeros anyway, wasting most of the array.

The idea

Overlay several sparse columns of the weight matrix into a single packed column of the systolic array. Because each original column is mostly zero, their non-zeros rarely collide — so one physical column can carry the work of many logical ones.

  • When two candidate weights do land in the same packed position, keep the one with larger magnitude (e.g. −6 kept over a smaller rival) and prune the other.
  • The packed cell's MAC multiplexes the correct input (say x₃ vs x₄) for the surviving weight.
  • Reported result: an 8× reduction in packed systolic-array size.
Vs. filterwise pruning

Compared with column-wise (filterwise) structured pruning, column combining allows a much more flexible pruning pattern — it keeps individual important weights that structured pruning would be forced to drop, then packs them densely. It can also be applied to each layer's input channels.

The training loop & row permutation

  • Column-combining pruning iterates: prune round i → combine → fine-tune → prune round i+1, jointly optimizing accuracy and packability.
  • Row permutation resolves collisions: reorder input rows so that large weights spread across packed columns. Two hardware realizations — Muxed (multiplexer selects the input) and Twisted (physically re-routed wiring).

04

Term quantization

Quantization shrinks models but low precision hurts. Term quantization (Kung, McDanel & Zhang, SC20) instead budgets the number of power-of-two terms — a finer knob than bit-width.

Why uniform low-precision fails

Take weights W = {1, 12, 5, 137} in 8-bit. Dropping to 4-bit simply truncates the low bits, giving {0, 0, 0, 128} — three of four weights vanish.

Root cause

Uniform quantization spends bits uniformly, but both weights and activations are highly biased in value — most are small, a few are large. Uniform low precision therefore incurs large quantization error exactly where the values live.

Values as sums of power-of-two terms

Any integer is a sum of power-of-two terms, and a product of two values is a sum of term-pair products:

12 = 2³ + 2² , 137 = 2⁷ + 2³ + 2⁰term decomposition
W₂X₂ = 12·2 = (2³+2²)·2¹ = 2³·2¹ + 2²·2¹term-pair products

Empirically, most quantized weight and data values need only 2–3 terms — so a "term budget" throws away far less signal than truncating bits.

The term budget

Set a group term budget: for a group of values, rank all terms by magnitude and keep only the largest few, dropping the small ones. This controls term-level computation directly.

W = {1, 12, 5, 137}W′ result
4-bit uniform quantization{0, 0, 0, 128}
TQ, budget = 4{0, 12, 0, 136}

At the same nominal cost, TQ keeps the 12 and reconstructs 137 far more faithfully (136 vs 128), because it spends its budget on the terms that actually carry magnitude.

HESE: fewer terms via signed powers Hybrid Encoding

To minimize the term count in the binary input, Hybrid Encoding for Shortened Expressions (HESE) allows signed power-of-two terms (a term may be subtracted). This yields a minimum-length expression and far fewer term-pair multiplications.

The tMAC: hardware for term-pairs

The term MAC (tMAC) implements TQ efficiently: it processes all term-pair multiplications across a group of weight and data values. Each term is just its exponent (2–3 bits), so a multiplication of two terms becomes an exponent addition, and accumulation reduces to half adders. tMACs tile into a systolic array fed by weight/data exponent queues, a weight index queue, and a term accumulator.

Error analysis

On ResNet-18, a group of 8 weights encoded with 14 terms = only 1.75 terms per weight on average. TQ adds a small error over 8-bit uniform, but achieves much lower error than 6-/7-bit uniform — 1.75 terms/value matches the quantization error of 5-bit uniform at a fraction of the compute.

Multi-resolution inference Zhang et al., ASPLOS 2021

Because the term budget is a dial, one model can run at different "resolutions." On a full battery use budget 60 (higher accuracy); on a low battery drop to budget 16 — trading hardware cost for accuracy along a smooth curve.

  • Train a meta multi-resolution model whose sub-models correspond to different term budgets (e.g. 2, 4, …, 10) — the terms are reusable, so a low-budget model is a prefix of a high-budget one.
  • Training uses knowledge distillation: each iteration jointly trains two sub-models, a teacher and a student, with loss L = L_T + L_S — cheaper than training all sub-models together.
  • Energy scales with the weight/data term budgets (α processed in γ cycles).
Result

Multi-resolution DNN incurs only 0.4%–3.8% degradation vs. the FP baseline (70.2%), while energy efficiency grows 3.25× on average as the term budget falls from 60 to 16.

05

Classic CNN accelerators

The canonical designs every accelerator paper cites. Each attacks the same enemy — data movement — from a different angle: dataflow, reconfiguration, or skipping zeros.

Diannao Chen et al., 2014

The first popular end-to-end DNN (CNN) accelerator. Synthesized at 65nm with Synopsys tools, hitting 482 GOP/s. Its Neural Functional Unit (NFU) is a three-stage pipeline:

  • NFU-1: multiplier units.
  • NFU-2: an adder tree that reduces the products.
  • NFU-3: the nonlinear unit (activation).

Input/output buffers NBin / NBout and synapse buffer SB stage operands, fed by DMA under a control processor.

Eyeriss row-stationary

Eyeriss optimizes the energy of the whole system — accelerator chip and off-chip DRAM — across varied CNN shapes by reconfiguring the architecture. Core: a spatial array of 168 PEs in a 12 × 14 rectangle, a 108-kB global buffer (GLB), an RLC CODEC, and a ReLU module.

Data reuse in a PE set

Row-stationary dataflow reuses three ways at once: filter rows stream horizontally across PEs, ifmap rows stream diagonally, and partial sums accumulate vertically. Reuse and accumulation inside the PE set cut accesses to the GLB and DRAM — where the energy actually goes.

Run-length encoding (RLC)

Eyeriss compresses input activations with RLC, exploiting the ~50% zeros ReLU produces. A stream like 0,0,12,0,0,0,0,53,0,0,22 is stored as (Run, Level) pairs in a 64-bit word:

[Run 5b | Level 16b] × 3 + [Term 1b] → (2,12)(4,53)(2,22)RLC word

Cnvlutin Albericio et al., 2016

A value-based accelerator: since a large fraction of CNN MACs are intrinsically ineffectual (one operand is zero), Cnvlutin skips them entirely. It encodes non-zero activations as a value list plus an offset list — [1,0,3] → values [1,3], offsets [0,2] — and indexes the matching weights, improving performance and energy with no accuracy loss.

06

Transformer accelerators

A transformer is still mostly matmul, so the systolic array carries over — but attention adds transposition, softmax, and layernorm, and these non-matmul operations are no longer negligible.

The self-attention dataflow

With batch B, sequence length L, embedding E, the block is a fixed sequence of shaped matmuls around a softmax:

  • LayerNorm the input X, then project to Q, K, V: (B,L,E)·(E,E) → (B,L,E) for each.
  • Scores: QKᵀ → (B,L,E)·(B,E,L) → (B,L,L) — the L² term.
  • Scale + Softmax: softmax(QKᵀ) → (B,L,L).
  • Context: softmax(QKᵀ)·V → (B,L,L)·(B,L,E) → (B,L,E).
  • Output projection (linear), then a residual add with the block input.

Operations other than multiplications

Beyond matmul, the accelerator must handle:

  • Transposition — needed to form Kᵀ and to re-lay-out tiles.
  • Nonlinear ops — Softmax, LayerNorm, and GeLU, all routed to the Special Function Unit.

Matmul is still the majority of latency, but nonlinear ops are not negligible (~30% combined), and "others" such as transposition take a large slice too. An accelerator that only speeds up matmul leaves nearly half the latency on the table — this is why transformer chips invest in the SFU and in data-layout hardware.

Tiling large matmuls

Real weight matrices dwarf the compute core, so the operands are partitioned into tiles sized to the array. A 512×256 by 256×256 product is broken into 128×128 tiles A₀₀, B₀₀, …, and the array sweeps tile-by-tile, accumulating partial results.

In-place matrix transposition

Forming Kᵀ naively needs a second buffer. In-place transposition rearranges a matrix within its existing memory, needing only minimal extra storage. Row-major [a b c d; e f g h] stored as abcdefgh becomes aebfcgdh.

How it works

The permutation decomposes into cycles — e.g. (b,c,e)→(e,b,c) and (d,f,g)→(f,g,d). Logic reads a few entries, permutes them, and writes them back, following each cycle. Finding swap patterns that minimize the number of permutations is a classic problem — Brenner's Algorithm 467 (1973) — and doing it with minimal memory-access cost is what the hardware optimizes.

Exam recap

Accelerator template

Compute core (Matrix + Vector MAC + SFU) fed by Weight/Activation SRAM. The whole game is keeping the MAC array fed without paying DRAM energy.

Systolic cell

z = w·x + y, v = x. Weight-stationary: w resident, data/partial-sums flow, inputs skewed so operands meet partial sums.

Double buffering

Ping-pong Mem₁/Mem₂ overlaps DRAM fill with array consume, hiding latency so the array never stalls.

Column combining

Pack sparse columns into one array column; keep larger-magnitude weight on collision. 8× smaller, 71.81% vs 69.0% structured.

Term quantization

Budget power-of-two terms, not bits. Values need ~2–3 terms; HESE uses signed powers; tMAC turns multiply into exponent-add.

Multi-resolution

One meta-model, reusable terms, distillation-trained. Dial budget 60→16 for 3.25× energy at 0.4–3.8% accuracy cost.

CNN classics

Diannao (first, NFU 3-stage) · Eyeriss (row-stationary, 12×14 PEs, RLC) · Cnvlutin (skip zero MACs, value+offset).

Transformer twist

Matmul ~55% latency, but softmax+layernorm+transpose ~45%. Tile big matmuls; transpose Kᵀ in-place via permutation cycles.
