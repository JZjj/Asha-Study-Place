Quantization Strategies for Efficient DNN Implementation

Trading precision for memory, compute, and energy — the number formats (INT / FP / BFP), the exact scale–zero-point math, where quantization error hides, and how to make the clipping range and rounding learnable.

NYU SAI Lab Spring 2026 INT · FP · BFP PTQ · QAT · STE PACT · QIL · QN
01

Why quantize, and the format landscape

Quantization replaces high-precision numbers (FP32) with a smaller set of low-precision values. Like pruning, the payoff is less storage and less compute / energy — but here we shrink each number instead of removing it.

Three families of data format

The lecture organizes everything around three basic representations, then two families of quantization method built on top of them.

FormatIdeaPrecision behaviourCompute
Fixed-point (INT)Uniform grid: one scale s maps integers to realsUniform — every step is the same sizeCheap integer adders / multipliers
Floating-point (FP)Sign · exponent · mantissa (IEEE-754)Relative — fine near 0, coarse when largeAlign/normalize; add is expensive
Block floating point (BFP)A group of values shares one exponent, keeps individual mantissasMiddle ground between INT and FPInner-group = fixed-point; cross-group = FP
INT vs FP — the one-line intuition

FP gives relative precision: under a fixed exponent the mantissa fills the gap evenly, but as the exponent grows the step size grows exponentially — small magnitudes get fine granularity, large magnitudes get coarse. INT gives uniform precision: every representable value is exactly the same distance s apart. Weights cluster near zero, so both stories matter.

02

Fixed-point (INT) quantization

Quantizing with a fixed-point format is called INT quantization. Two hyperparameters define it: the clipping range (often symmetric (−L, L)) and the bitwidth b. Everything downstream is arithmetic on the integer code xint.

Symmetric quantization round-trip

To convert a real number x to its INT representation and back:

s = 2L / (2b − 2)scale (step size)
xc = Clip(x, L, −L)clip into [−L, L]
xint = round(xc / s)integer code
xq = s · xintrescale (dequantize)
Why 2b − 2 and not 2b − 1

With s = 2L/(2b−2) the grid is symmetric and zero is exactly representable — important because zeros appear constantly (padding, ReLU, pruned weights). Using 2b−1 gives one more level but no exact zero. The representation power is uniform inside the clipping range, and all computation runs on xint.

Worked example

X = [1.1, 2.4, −0.3, 0.8], bitwidth b = 3, L = 2.

  • Scale: s = 4 / (2³−2) = 4/6 = 2/3
  • Clip: x_c = [1.1, 2, −0.3, 0.8] (the 2.4 is clamped to L = 2)
  • Integer code: x_int = round(x_c / s) = [2, 3, 0, 1]
  • Dequantize: x_q = s·x_int = [1.33, 2.0, 0.0, 0.67]

The gap between x_q and the original X is the quantization error; clamping 2.4 → 2.0 is truncation (clipping) error.

Asymmetric (unsymmetrical) quantization INT8-style

When the range isn't centred on zero, use an explicit (Lmin, Lmax) and add an offset (a zero-point) back at the end:

s = (Lmax − Lmin) / (2b − 1)scale
xint = round((xc − Lmin) / s)shift, then round
xq = s · xint + Lminrescale + offset

Example with the same X, b = 3, Lmax = 2, Lmin = −0.5: s = 2.5/7 = 0.357, x_int = [4, 7, 1, 4], x_q = [0.93, 2.0, −0.14, 0.93]. The asymmetric range packs the grid tighter around the data, but the offset Lmin reappears in every arithmetic rule.

Computing on quantized values

The whole point of INT is that math can stay in integers. What survives depends on whether the two operands share a scale.

OpSame scale s (symmetric)Different scales sx, sy
Add / Subs(x_int ± y_int)Hard to compute (scales don't factor out)
Multiplys²(x_int × y_int)s_x s_y (x_int × y_int)
Dividex_int / y_int(s_x/s_y)(x_int / y_int)

For the asymmetric case the offset makes even addition messy, and multiplication grows cross terms:

xq + yq = s(xint + yint) + 2Lminadd (subtraction drops the offset)
xq·yq = sxsy(xint·yint) + Lmin,xyqsy + Lmin,yxqsx + Lmin,xLmin,ymultiply — extra terms
Design consequence

Symmetric quantization keeps the hardware simple (one scale factors out cleanly). Asymmetric fits skewed distributions better but drags an extra offset through every operation, and division is essentially impractical to implement. This is why most weight quantization is symmetric.

Dot products and the INT-MAC pipeline

For a dot product, all elements within a tensor share one scale, but the scale can differ across tensors. The shared scale pulls straight out of the sum:

xq,1yq,1 + xq,2yq,2 = sxsy(xint,1yint,1 + xint,2yint,2)quantized dot product

So an INT layer is a three-stage pipeline — quantize → INT MAC → dequantize — versus a single FP MAC in full precision. The middle stage runs on cheap integer hardware; the outer stages are the tax you pay for it.

Granularity: per-tensor vs per-row vs per-column

INT can be applied to a whole tensor or to smaller blocks, each with its own scale.

Per-tensor

One scale for everything. Cheapest, coarsest — a single outlier stretches the range for all values.

Row-wise

A scale per row. In the slide's example this gives low error — rows are more homogeneous.

Column-wise

A scale per column. Here it gives high error — the grouping doesn't match the data's structure.

The catch

A higher (finer) quantization granularity lowers error but incurs conversion overhead — more scales to store, compute, and apply. Granularity is the error-vs-overhead knob, and which axis (row vs column) wins depends entirely on how the data is laid out.

Extreme low bits: binary, ternary, and logarithmic

  • Binary quantization maps weights to {−1, +1}; ternary to {−1, 0, +1}.
  • Both give multiplication-free DNNs — a multiply by ±1 is just a sign flip, so the expensive multiplier disappears.
  • Logarithmic arithmetic is a specialized INT scheme using only power-of-two values. Each number is stored as its exponent = log(INT); 8 distinct values need only 3 bits, and multiplication becomes bit-shifting / adding exponents (a×2 = a≪1). Cheap and cost-effective in hardware.
03

Floating-point & block floating point

FP trades the uniform INT grid for a sign / exponent / mantissa layout that gives finer resolution near zero — at the cost of expensive addition. BFP sits between the two.

IEEE-754 floating point

An FP number has three fields — sign s, exponent e, mantissa m:

x = (−1)s × 2e−bias × (1 + m)m = (0.b₀b₁…b₂₂)₂
  • FP32: 1 sign · 8 exponent · 23 mantissa, bias = 127.
  • FP16: 1 sign · 5 exponent · 10 mantissa.
  • The exponent field is unsigned (hence the bias); an all-zero bit stream is the special case for 0.

To encode a real number: x = |x|, s = sign(x), a = ⌊log₂ x⌋, e = a + bias, m = x/2ᵃ − 1. For example x = −13.24, bias = 127 → a = 3, e = 130, m = 0.655, i.e. s = 1, e = (10000010)₂. And 5.5 = (−1)⁰ × 2^(129−127) × (1.011)₂.

Where FP's precision goes

In the formula, the 2e−bias factor sets the scale and the (1+m) factor fills the interval evenly. So for a fixed exponent the mantissa spreads points uniformly, but each time the exponent ticks up the whole interval doubles — granularity decreases exponentially with magnitude. That's why FP has "better representation power for small magnitudes."

Computing with FP add is the hard one

  • Add / Sub: you must first align the exponents (shift one mantissa), then add — an extra alignment step every time.
  • Multiply: simpler — sum the exponents, multiply the mantissas: e = e₁ + e₂, 1+m = normalizer(1.m₁ × 1.m₂).
  • Takeaway: addition/subtraction is expensive for FP (alignment + normalization), the opposite of INT where addition is trivial.

Customized FP formats for DNNs

Many narrow FP formats trade exponent bits (dynamic range) against mantissa bits (precision):

FormatExponentMantissaNote
FP32823IEEE-754 baseline

HFP84 or 53 or 28-bit float, two variants

Like INT, FP can also be applied to a block of numbers with a customizable block size — which leads directly to BFP.

Block floating point (BFP)

BFP applies a shared exponent to a group of fixed-point values, balancing precision and dynamic range while cutting the cost of full FP. A group of size g keeps one group exponent (with a bias shared across groups) plus per-element sign + mantissa.

FP: x = (−1)s × 2e−bias × (1 + m)has a leading 1
BFP: x = (−1)s × 2e−bias × mm = (b₀.b₁b₂…)₂ — no leading 1
  • Inner-group operations run as fixed-point arithmetic (shared exponent → integers).
  • Cross-group operations run as floating-point (different exponents must be reconciled).
  • Encoding a block: find the max value → set the group exponent → shift all significands to that exponent → store sign + mantissa. Example: {5.5, 2.625, −3.125, 2.75} → max 5.5 → group exponent 2 → each value re-expressed as (−1)ˢ×2²×(0.…)₂.
Why BFP is attractive

One exponent amortized over a whole block means near-FP dynamic range at near-INT compute cost. Formats like MSFP-12 (e.g. g=16, e=8, m=3) are exactly this middle ground — the reason BFP shows up in modern accelerators.

A taxonomy of quantization methods

Quantization techniques can be classified along five independent perspectives. A real method is a point in this space.

WhatWeight quantizationActivation quantization
WhenPost-training quantization (PTQ)Quantization-aware training (QAT)
GranularityTensor-basedVector-based · Group-based
PurposeQuantize for inferenceQuantize for training
RuleDeterministicStochastic

Weight quantization — the clip range matters most

Weights follow a Gaussian-like distribution. Outliers in the tail blow up the quantization error, so choosing the clip range L is critical:

  • Small L: large truncation error (many values clipped) but low error for small values (grid is dense near zero).
  • Large L: small truncation error but large error for small values (grid is spread thin, and most weights are small).
  • Common heuristics: L = 0.9 × max(|W|) or L = 0.95 × max(|W|) — the constants are chosen by experience (and §6 asks: can we learn them instead?).

Activation quantization — the dynamic tax

Activations depend on the input, so their quantization must be done dynamically, adding compute overhead. They also pass through nonlinear functions (BatchNorm, ReLU) that are sensitive to quantization error, so a layer dequantizes back to FP16/32 around them: FP2INT → INT Conv → INT2FP → BatchNorm → ReLU → FP2INT → …

Low precision can cost more than it saves

On a 4090 GPU a 577×1024 · 1024×1024 projection layer measured (ms): FP32 = 0.078, FP16 = 0.052, FP8 = 0.187, FP8 w/o scale = 0.132, INT4 = 0.136. The low-precision paths are slower because the quantize/dequantize machinery dominates — roughly Cal_scale 27% · Quant 26% · MatMul 25% · Dequantize 22%. For low-precision, the conversion overhead can exceed the MAC savings.

Deterministic vs stochastic quantization

Consider a = 0.2 on a {0, 1} grid. Deterministic rounding gives q(a) = 0 every time — a systematic bias. Stochastic quantization instead rounds probabilistically:

q(a) = 1 with p = 0.2, 0 with p = 0.8unbiased in expectation
  • In expectation E[q(a)] = 0.2 = a — the bias vanishes on average.
  • For QAT the deterministic bias isn't fatal because BatchNorm's own bias absorbs it.
  • Stochastic quantization is especially useful for accelerating DNN training, where unbiased gradients matter.

Clustering-based quantization non-uniform

Quantization and clustering are cousins: clustering assigns each value to a centroid; quantization maps each value to a predefined level. K-means-style clustering picks centroids freely (non-uniform levels).

Trade-off

Clustering compresses the weight matrix well and stores efficiently (indices + a small codebook), and its flexible centroids usually give better accuracy — but the irregular levels are hard to accelerate for computation. Good for storage, awkward for MACs.

05

QAT internals: STE & quantized-gradient training

PTQ is cheap but lossy; QAT bakes quantization into training for higher accuracy. Its central problem: the quantizer is a staircase whose derivative is zero almost everywhere — so how does the gradient get through?

PTQ vs QAT

PTQ

Train in full precision, then quantize the weights once. Lower computational cost, lower accuracy.

QAT

Quantize weights/activations, train, repeat until convergence. Higher accuracy, higher cost.

When to use which

For models that are expensive to train (LLMs), retraining under QAT is infeasible, so PTQ is the default — you accept a bit of accuracy loss to avoid another training run.

The backward-pass problem

Insert a quantizer Q so W → Q → W′, then Y = W′A, Z = ReLU(Y). Backprop needs the full chain:

∂ℒ/∂W = (∂ℒ/∂Z)(∂Z/∂Y)(∂Y/∂W′)(∂W′/∂W)the troublesome factor is ∂W′/∂W

Since W′ = Q(W) is a staircase, ∂W′/∂W = 0 almost everywhere — the gradient dies and the network becomes untrainable.

Straight-Through Estimator (STE)

STE simply pretends the quantizer is the identity on the backward pass:

∂W′/∂W = 1Straight-Through Estimator
The rule in one sentence

Forward pass: apply quantization. Backward pass: ignore it. The forward pass sees the quantized weights (so the network learns to cope with quantization), while gradients flow through as if the quantizer weren't there. (Li, Hao et al., 2017.)

STE is the crudest estimator of the sign/staircase gradient. Smoother surrogates keep more signal — Bi-Real Net (Liu et al., 2020) replaces the hard Sign(x) with Clip(−1, x, 1), ApproxSign, or a 3-order-ApproxSign, each giving a non-zero, better-shaped derivative near the step.

PyTorch shape of a quantized layer

A quantized conv layer just wraps the weight in a quantizer Q before the matmul:

  • L = 0.9 * w.abs().max() — pick the clip range from the weights.
  • w = torch.clip(w, -L, L) — clamp.
  • scale = 2L / (2**b − 2) — the symmetric scale.
  • wq = (w/scale).round() * scale — quantize then dequantize; STE handles the .round() gradient.

Quantizing during training — mind the gradients

Training has three matmuls, not one: forward Y = WA, data-gradient ∇X = ∇Y · Wᵀ, and weight-gradient ∇W = Xᵀ · ∇Y. To accelerate training you quantize the operands of all three with Q(·) (or a stochastic SQ(·)).

Gradients resist quantization

The gradient is much more sensitive to quantization error than weights or activations. DNN gradients are near-lognormal (Chmiel et al., 2020) — a huge dynamic range concentrated near zero with a long tail — which is exactly the distribution a uniform INT grid handles worst. This is why stochastic quantization and careful formats are needed for training-time quantization.

06

Learnable & adaptive quantization

The clip range L was picked by a hand-tuned constant (0.9, 0.95). Learnable quantization makes the quantization hyperparameters — the range, the rounding, even the whole mapping — trainable via gradient descent.

PACT — learn the clipping range

PACT (Choi et al., 2018) makes the clip level l a learnable parameter. Clip, then quantize:

Clip(x, l) = l if x ≥ l; x if −l ≤ x ≤ l; −l if x ≤ −lparameterized clip
∂ℒ/∂l = (∂ℒ/∂xq)(∂xq/∂xc)(∂xc/∂l) ≈ (∂ℒ/∂xq)(∂xc/∂l)STE drops ∂x_q/∂x_c ≈ 1

The gradient of the clip w.r.t. l is 1 where the input is saturated (|x| ≥ l) and 0 in the pass-through region — so l only feels pressure from the clipped tail, letting the network learn how tight to squeeze the range.

QIL — Quantization Interval Learning

QIL (Jung et al., 2019) learns an asymmetric interval (c−L, c+L) and adds rounding flexibility by inserting a learnable transform F before quantization:

wq = Q(w) → wq = Q(F(w))transform, then quantize
ŵ = 0 if |w| < cW−dW; sign(w) if |w| > cW+dW; (αW|w|+βW)γ·sign(w) otherwiselearnable mapping F
  • The γ exponent bends the mapping: γ = 1 is linear, γ < 1 and γ > 1 curve it — giving control over which FP weight rounds to which level (e.g. w = 0.2 can be mapped to ŵ = 0.8 before rounding).
  • w_q = Q(F(w)) is precomputed and stored for inference after training.
  • QIL is applied to weights, not activations — the mapping is too much computational overhead to run dynamically on activations.

Quantization Networks (QN) — quantization as a differentiable function

QN (Yang et al., 2019) reframes low-bit quantization as a differentiable non-linear function — a sum of soft step functions:

y = α( Σi=1..n si 𝒜(βx − bi) − o ), 𝒜(x) = 1 if x ≥ 0 else 0n+1 quantization intervals
  • n + 1 = number of quantization intervals; β scales the input; s_i, b_i are the scales and biases of each unit step.
  • To make it trainable, replace the hard staircase 𝒜 with a sigmoid σ(Tx) = 1/(1+exp(−Tx)) and progressively increase T during training — the function anneals from a smooth ramp (T=1) toward a sharp staircase (T=50, "complete quantization"). Related: differentiable soft quantization (Gong et al., 2019).

The deeper problem: quantization ignores the input

The major drawback of naive quantization is that it doesn't consider the impact of the input when deciding how to round. A small weight that always multiplies a large, frequent activation can matter more than the rounding error suggests — and per-weight quantization can't see that.

Calibration-based reconstruction

The fix: use a calibration dataset, profile the real inputs/outputs Xl, Yl at each layer, and solve for the quantized weights that best reproduce the layer's output rather than best approximate the weights themselves:

minWl ‖ Yl − Xl Q(Wl) ‖² for each layer loutput-aware, layer-wise

This output-reconstruction objective is the principle behind modern PTQ methods for LLMs — optimize what the layer computes, not just what it stores.

★

Exam recap

INT vs FP

INT = uniform steps; FP = relative precision (fine near 0, coarse when large) via 2e−bias×(1+m). BFP shares one exponent per block.

Symmetric INT

s = 2L/(2ᵇ−2), x_int = round(clip(x)/s), x_q = s·x_int. The −2 keeps zero exactly representable.

Asymmetric INT

s = (L_max−L_min)/(2ᵇ−1); add offset L_min back. Fits skewed data but drags an offset through every op; division impractical.

Compute rules

Same scale: add = s(x+y), mul = s²(xy). FP: multiply is easy (sum exponents), add needs exponent alignment.

Granularity

Per-tensor → row → column: finer = lower error but conversion overhead. Which axis wins depends on data layout.

Clip range L

Weights are Gaussian-ish; L=0.9–0.95·max(|W|). Small L = truncation error, large L = error on small values. PACT learns L.

PTQ vs QAT + STE

PTQ cheap/lossy (used for LLMs); QAT retrains. STE: quantize forward, ∂W′/∂W = 1 backward.

Gradients are hard

Near-lognormal, very sensitive to quant error → use stochastic quantization; deterministic bias absorbed by BN.

Learnable methods

PACT (learn L), QIL (learn the mapping F, w_q=Q(F(w))), QN (soft steps annealed via σ(Tx), T↑).

Activation cost

Dynamic quantize/dequantize overhead can exceed MAC savings — low precision was slower than FP16 on a 4090.

Extreme low bit

Binary {−1,1} / ternary {−1,0,1} = multiplication-free. Log arithmetic = power-of-two, multiply → bit-shift.

Calibration

Quantization ignores the input; fix with min‖Y−X·Q(W)‖² per layer on a calibration set — output-aware reconstruction.
